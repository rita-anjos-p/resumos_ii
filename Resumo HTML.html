O que é HTML?

HyperText Markup Language
É a linguagem usada para escrever docs web.

  • Define o conteúdo de docs web.
  • Define a estrutura desse mesmo conteúdo
  • Define como se interligam docs web hipertexto

Um documento HTML é composto por elementos html. Estes são usados para estruturar o texto, tendo cada um um significado associado (ex: título, parágrafo, etc). E segue uma estrutura fixa.

Elementos HTML

Cada elemento é constituído por 3 partes:

  • a marca de início: <elemento>
  • o conteúdo: <elementgtt; conteúdo
  • a marca de fim (igual à de ínicio mas precedida de uma barra: <elemento>conteúdo</elemento>

A maneira como o elementos é escrito (maiúsculas ou minúsculas) é indiferente visto que se trata apenas de um código que específica o que é o conteúdo. No entanto o conteúdo é sensível a essas alterações, visto que é o que acaba por ser mostrado ao leitor, tal e qual foi escrito.
A marca de início de um elemento pode ter vários atributos associados. Há atributos obrigatórios e outros opcionais. Quando um elemento tem um atributo associado este não aparace na marca de fim, apenas na de início.
Vários espaços em branco seguidos não são contabilizados e convertem-se em apenas um só espaço na página web.

Tipos de elementos

Há dois tipos de elementos em HTML:

  • Elementos não vazios: têm conteúdo
    • ex: <h1>título 1</1>
  • Elementos vazios: não têm conteúdo nem marca de fim
    • ex: <img src="fotografia">

Organização de elementos

Os elementos devem ser organizados de maneira a que estejam encaixados, uns dentro dos outros, e não intercalados. Isto é: um elemento 2 que começe dentro de um elemento 1 previamente iniciado deve ser finalizado antes que o elemento 1 o seja.
Entende-se então que: <1> <2> </2> </1> está correto. E que: <1> <2> </1> </2> já não irá funioncar, causando erros.

Estrutura de um documento HTML

Verificação de erros

Todos os documentos html devem seguir a mesma estrutura, normalizada pela W3C (World Wide Web Consortium). Esta organização foi criada para garantir a compatibilidade e entendimento entre empresas no desenvolvimento e adoção de tecnologias web. É através do seu serviço automático de avaliação da conformidade do código que se pode verificar erros cometidos ao longo dos documentos criados. O link é: http://validator.w3.org/. É boa prática ir verificando o documento ao longo da criação do código visto que torna mais fácil a sua correção.

Base de um documento HTML

O documento HTML tem como base a seguinte estrutura:
   <!DOCTYPE html>
       <html>
          <head>
             <title>Título do Documento</title>
          </head>
           <body>
             <p>Corpo do Documento</p>
           </body>
      </html>
Ou seja, começa-se por identificar o tipo de documento, html é raiz do documento, defini-se o head (cabeçalho) com informações gerais, necessárias ao navegador para processar e apresentar o doc. Após fechar o head, abre-se o body onde ficaram todas as informações que devem constar na página propriamente dita. No final fecha-se o html.

Caracteres complexos

A maioria dos caracteres que utilizamos são definidos por 6 bytes enquanto caracteres especiais como "ç" ou qualquer caracter acentuado já utiliza mais bytes. Para que o texto apareça corretamente quando for lido é necessário adicionar "<meta charset=“utf-8” />" uma linha abaixo da definição do tipo de documento.

Definição da língua - optativo

A definição da língua em que a página vai ser escrita é optativo, no entanto é aconselhado visto que faz parte dos warnings dados pelo W3C aquando da verificação do documento. Par isso basta adicionar a <html> o atributo: lang="língua". O elemento, na sua totalidade e supondo que a língua é o português, deve então ficar: <html lang="pt">. No final do documento, ao fechar o elemento html não há necessidade de repetir o atributo (tal como acontece em todos os casos).

Lista de elementos HTML e seus atributos

Cabeçalho

head - define o que faz parte do cabeçalho.
Obrigatoriamente, dentro de head tem que estar o:
title - é normalmente apresentado no topo da janela do navegador.
Como optativos pode-se ter:
meta - permite especificar informações sobre o documento, ou
link - permite estabelecer ligações entre o documento e outros recursos, nomeadamente folhas de estilo (CSS).

Corpo

O corpo do documento é definido pelo elemento body. Todos os próximos elementos mecionados devem aparecer subordinados a este.

Títulos

Há 6 níveis de título em html. Estão denominados por ordem de importância, sendo o mais importante h1, e o menos importanto o h6.
h1 - Título principal
h2 - Título secundário
h6 - Título de menor importancia
Cada navegador web têm estilos pre-definidos, por omissão, para cada um destes elementos, que podem ser alterados através de folhas de estilo.

Parágrafos, Destaques e Quebras de linha

p - parágrafo. Delimita blocos de informação, textual ou não. Fehcar <p> é opcional (por ser subentendida com a abertura do próximo <p>)
strong - destaca o texto (tipo bold).
em - enfatisa o texto (tipo itálico).
br - quebra de linha. É um elemento vazio

Listas

Há dois tipos de listas em html: listas ordenadas, que são numeradas, e listas não ordenadas, que apresentam apenas pontos para cada entrada.
ol - orderly list
ul - unorderly list
li - cada item dentro da lista deve ser apresentado dentro deste elemento, para que seja identificado.
exemplo:
<ol>
    <li> item 1 </li>
</ol>

Tabelas

table - delimita a tabela
tr - table row, especifica cada linha
td - table division, especifica cada célula dentro de cada linha
th - table header, define células tipo título (bold)
Atributos de TD ou TH:
rowspan - agrupa células (número deve ser especificado) por linha
colspan - agrupa células (número deve ser especificado) por coluna
exemplo:
<table>
    <tr>
        <th rowspan="2"> Célula 1 </th>
        <td> Célula 2 </td>
    </tr>
    <tr>
        <td> Célula 3 </td>
    </tr>
</table>
cujo resultado seria:

Célula 1 Célula 2
Célula 3

Imagens


img - imagem
Atributo obrigatório:
src - source. Indica o local da imagem em relação ao documento. O caminho até aos ficheiros é feito utilizando a mesma lógica que os comandos linux.
Atributos opcionais:
alt - texto alternativo. Não é obrigatório mas é boa prática visto que viabilizam a compreensão de imagens por pessoas com dificuldades de visão, facilita a indexação dos conteúdos por parte dos motores de pesquisa e em navegadores que não apresentem imagens passa a existir uma maior compatibilidade.
height - altura.
width - largura. Usar a altura e a largura permite o navegador apresentar o resto do conteúdo deixando o espaço adequado para as imagens que demorem a carregar. Cada vez mais especificado através do CSS.

Ligações


a - anchor. permite establecer ligações do doc a outros recursos da web
href - hyper reference. permite indicar o destino da hiperligação.
Ligações podem ser absolutas, quando o destino é independente do local do documento html, ou relativas, para referenciar documentos do computador tendo como ponto de partida o próprio documento html.
exemplos:
referência absoluta:
<a href="website"> WEBSITE </a>
cujo resultado seria:
WEBSITE
referência relativa: a um documento adjacente a este
<a href="Doc_exemplo.pdf"> Documento referência relativa </a> (neste caso só o nome do documento porque encontram-se na mesma pasta)
cujo resultado seria:
Documento referência relativa
NOTA: esta referência só funciona no computador em questão
referência no próprio documento: como acontece no primeiro item do índice:
<a href="#o_que_e_html"> O que é HTML? </a>

Elementos Estruturais

header - define o cabeçalho
footer - delimita o rodapé
nav - define a estrutura de navegação da página através de links (como menus, tabelas de conteúdo, ...)
article - é usada para definir partes do conteúdo da página de possam ser usadas independentemente
section - serve para dividir o documento em diferentes secções
div - division. Não tem significado semântico associado, permite agrupar elementos do doc. É usado em conjunto com CSS para definir zonas dos docs.
id - identifier. Atributo de div. Serve para distinguir diferentes divisores.
exemplo:
<div id="grupo1">
    <h1>Título 1</h1>
       <p>Isto é o grupo 1.</p>
</div>

Comentários

Utilizando <!-- comentário --> , temos a oportunidade de inserir a meio do documento um comentário que é ignorado no navegador web.

Organização de sítios web

"Um sítio web corresponde ao conjunto de documentos e recursos necessários para apresentar um sitio"

Nomeação de ficheiros

Ao nomear ficheiros é importante evitar caracteres especiais, acentos ou espaços, preferivelmente evitar também capitalização de letras.
Por norma, o ficheiro base de um website deve chamar.se "index.html"

Organização de ficheiros

Manter os ficheiros organizados por tipo e para cada secção do website é aconselhado para facilitar trabalho.

Vantagens de uma boa organização

Ao usar bons nomes para os ficheiros e ao manter uma organização simples e eficiente, o resultado serão URLs fáceis de ler, com entendimento do seu conteúdo mais rápido.
Por exemplo: "http://www.google.com/images/logo.gif" versus "http://sigarra.up.pt/web_base.gera_pagina?p_pagina=2436"
